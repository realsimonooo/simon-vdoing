# 01. Web 基础

## 🔗URL / URN / URI

+ 代表三个标准：Locator / Name / Identifier ；

### 🔗URL

+ URL 以文字形式表示获得网络**资源**的方法；

+ URL 的格式： `<scheme>:<scheme-specific-part>`

  + `scheme` / 协议：指定获取资源的方式:

    + FTP： 文件传输协议
    + HTTP： 超文本传输协议
    + Mailto：📧电子邮件
    + File：特定主机文件名

  + `scheme-specific-part` 协议特定部分依据协议而定；

    格式： `<用户>:<密码>@<主机>:<端口号>/<文件路径>` 



## 📃HTTP

+ 架构在TCP/IP之上的应用层协议；

+ 是一种通信协议：两台计算机的联系方式；

  + SMTP： 发送信件；
  + FTP：传输文件；
  + POP3：下载新建；
  + HTTP：Web网站之间的沟通；

+ 特性：

  + Request/Response模型 （请求/响应）

    >浏览器发出对Web网站取得资源的请求，Web网站将该请求要求的资源响应给浏览器；
    >
    >HTML5支持Server Sent Event，Web网站一直持续处于请求的状态；

  + Stateless协议 （无状态）

    > HTTP协议下，Web网站响应客户端后，没有获取客户端的信息；

### HTTP请求方法

#### `get`请求

向Web网站获取**制定的**资源，发出请求时的一并告诉给Web网站：

1. 所请求资源的URL； 

   请求参数：参数名称和参数值

   `/download.do?file=servlet&user=caterpillar`

2. 标头（Header）的信息；

   `user-agent` 客户端/浏览器版本信息；

   `accept-language`浏览器语言

特点：

+ 请求可以发送的参数**长度有限**；

+ 不适合用于大量数据请求；

+ 通常浏览器的地址栏会显示参数信息；

#### `post`请求

特点：

+ 适用于大量或者复杂的数据信息（文件上传）；

+ Post请求将请求参数移到了信息体的最后，请求参数（信息体）的内容长度不受限制；（敏感信息，密码）

  > 非加密联机还是会被第三方获取；

## Servlet / JSP

### 🥣Web容器

+ Web容器是Servlet/JSP唯一认得的HTTP服务器，介于实体HTTP服务器和Servlet之间；

  + ～ JVM是JAva程序唯一认识的操作系统，可执行文件为`.class`，介于实体系统和Java程序之间；

+ Web容器用来持有、保存对象的集合对象；并且负责对象的生命周期♻️以及相关服务的连接🔗；

  > + 具体，容器是Java写的程序，运行于JVM上，不同容器负责不同的工作；
  >
  >   Web容器也是Java写的程序；
  >
  > + 抽象层面，将Web容器视为运行Servlet/JSP的HTTP服务器；
  >
  >   只要Servlet/JSP撰写时符合Web容器的标准规范，就可以在不同厂商实现的Web容器上运行✅。

+ Web容器的剖析与转换：HTTP文字性的通信协议将如何转换为Servlet/JSP中可用的Java对象（`HttpServletRequest, HttpServletResponse`）；

+ **Web容器**：真正负责请求、响应的是Servlet；

  1. 浏览器对HTML服务器发出请求；

  2. 将请求转由**Web容器处理**；

     > 剖析HTML请求内容，创建各种Java对象；

  3. Web容器由请求的**URI**决定要使用哪个Servlet来处理请求；

     > 定义URI与Servlet的对应关系；

  4. **Servlet**根据请求对象的信息决定如何处理，通过响应对象来创建响应；

  5. Web容器与HTTP服务器沟通，**HTTP服务器**将相关响应对象转换为HTTP响应并传回给浏览器；

+ Servlet执行在Web容器之中，Web容器由服务器上的JVM启动，JVM本身就是服务器上的一个可执行程序，当一个请求来到时，Web容器为每个请求分配一个线程。

  > 有多个请求时，将有多个线程来处理；（JVM）
  >
  > Web容器可能会使用同一个Servlet实例来处理多个请求，相当于多个线程共享同一个对象。因此需要注意线程安全问题，避免数据错乱；

+ 各平台不同的容器：

  + EJB -> EJB容器；
  + 应用程序客户端 -> 应用程序客户端容器；

  当不理解容器的行为就容易引发程序执行的问题，安全弱点；

### Servlet与JSP的关系

+ JSP会被Web容器转换为Servlet的`.java`源文件、编译为`.class`	文件，最终提供服务的还是Servlet实例。

  > JSP和Servlet是一面两体；
