# 操作系统

操作系统是**计算机系统中的一个系统软件**；

+ 用于管理和控制计算机所有的硬件和软件资源；

  （是硬件与用户软件之间的中间件程序）

+ **接口**：计算机硬件与其它软件的接口；

  用户和计算机的接口；

+ 主要功能：处理机管理，存储管理，设备管理，信息（文件）管理；

  （操作系统是一个资源管理程序、控制调度程序）；

  （操作系统一般由一个内核程序，开机后一直运行）；



## ==现代操作系统的特点==

1. 并发行
2. 共享性
3. 虚拟性
4. 不确定性



## 操作系统的发展

### 批处理系统

+ 作用：自动、成批地处理用户的作业；
+ 特点：成批提交程序、数据以及操作说明给系统直到作业运行完毕再进行交互，根据输出结果分析运行情况；

#### 联机批处理

+ 优点：解决了作业的自动转接，减少了作业建立和人工操作时间；
+ 缺点：在作业输入和结果输出过程中，**主机CPU处于停止等待状态，浪费了时间资源**；

#### 脱机批处理

+ 特点：增设一台卫星机
  + 不与主机相连、专门用于输入输出设备；
+ 优点：主机与卫星机并行工作，主机不直接与输入输出设备打交道，充分发挥了主机的高速度计算能力；

### 多道程序系统

单道运行：每次只调用**一个用户程序**进行内存并运行；

多道程序运行特点：

1. 多道： 内存同时存放多道相互独立的程序；
2. 宏观上并行：各道程序运行过程中，先后开始各自的运行，但都未运行完毕；
3. 微观上串行：各道程序轮流使用CPU，交替执行；

<img src="https://cdn.jsdelivr.net/gh/realsimonooo/image-repo@master/blog/image-20210304162024776.png" alt="image-20210304162024776"  />

### 分时操作系统

+ 作用： 各终端用户共享主机和外部设备；
+ 实现：将CPU的时间划分成**时间片**，用于**==轮流==**接受和处理命令；
+ 特点：提高CPU利用率；提高用户CPU交互速度；

### 实时操作系统

+ 用于工业过程控制、军事实时控制；
+ 特点：处理外来信息足够快，对于被控对象响应快速（秒级、毫秒级）

### 通用操作系统

+ 特点： 同时兼备多道批处理、分时、实时处理的功能或其中两种以上的功能；
+ 实时批处理系统：
  + 实时任务：前台作业
  + 批作业：后台作业
+ 分时批处理系统：

### 批处理操作系统

是现代操作系统的趋势，大多都具有批处理功能；

<img src="https://cdn.jsdelivr.net/gh/realsimonooo/image-repo@master/blog/NWGgCo.png" alt="image-20210306145107580" style="zoom:67%;" />

+ 特点：
  1. 用户脱机使用计算机；
  2. 作业批处理；
  3. 多道程序运行；
+ 优点：
  + 系统资源为多个作业共享，作业之间自动调度执行；
  + 运行过程中用户不干预自己的作业，提高系统资源利用率和作业吞吐量；
+ 缺点：
  + 无交互性，用户运行控制能力 ；
  + 批处理时作业周转时间长，用户使用不方便；

+ 分别多道程序系统（muiltiprogramming）和**多重**处理系统（multiprocessing）；

  <img src="https://cdn.jsdelivr.net/gh/realsimonooo/image-repo@master/blog/F5kvyO.png" alt="image-20210306145932750" style="zoom:67%;" />

  1. 多重处理系统配置多个CPU，从而可以执行多道程序；
  2. 要有效地使用多重处理系统，必须采用多道程序设计技术；但反之，多道程序设计技术不一定要求多重处理系统的支持；
  3. 多重处理系统优点：提高系统吞吐量、可靠性、计算能力、并行处理能力；

### 分时系统

采用时间片轮转的方法，使一台计算机为多个终端用户服务，

1. 交互性；（调试和运行程序）

2. 多用户同时性；

3. 独立性；（快速响应的基础上）

4. (多道程序设计)

   ![image-20210306151447531](https://cdn.jsdelivr.net/gh/realsimonooo/image-repo@master/blog/CyMCcQ.png)

### 实时系统

是另外一类联机的操作系统，对外部的请求，实时操作系统能够在规定的时间内处理完毕。专业性，**与批处理系统和分时系统相比，实时系统的资源利用率可能较低**；

+ 特点：

  1. 即时响应性；（有效的等待时间和响应时间）

  2. 高可靠性；

  3. 用户控制；

  4. 系统出错时处理能力强；

+ 设计时需要考虑的因素：

  1. 实时时钟管理；（定时处理、延时处理）
  2. 连续的人-机对话；
  3. 过载保护；
  4. 高度可靠性和安全性需要采取冗余措施；

### 通用操作系统

多种类型操作特征，同时兼有批处理、分时、实时处理和多重处理的功能；

#### 个人计算机操作系统

高速数据处理能力和实时多任务操作系统；

#### 网络操作系统

+ 是互联的计算机系统群体，在物理上分散；
+ 是自治的；
+ 系统互联通过通信设施（硬件、软件）实现；
+ 系统通过通信设施执行信息交换、资源共享、互操作和写作处理，实现多种应用要求；

#### 分布式操作系统

分布系统通过通信网络将物理上分布、具有自治功能的数据处理系统或计算机系统互连起来，实现信息交换和资源共享，协作完成任务；

+ 特点： 
  1. 功能的分布；
  2. 坚强性；
  3. 高可靠性；

+ 区别于网络系统：

1. 没有标准的协议；（网络也可认为是一种分布式系统）
2. 统一的操作系统；
3. 系统的透明性

#### 多道批处理系统

+ 特点
  1. 多道：内存中同时存放几个作业；
  2. 宏观上并行；
  3. 微观上串行；
+ 优点：
  1. 资源利用率高（CPU和内存）
  2. 作业吞吐量大；
+ 缺点：
  1. 用户交互性差；
  2. 作业平均周转时间长；

#### 通道和中断技术

+ 通道： 

  1. 用于控制I/O设备与内存之间的数据传输；独立于CPU运行，实现CPU与I/O的**并行**；
  2. 有专用的I/O处理器
  3. 实现I/O联机处理

## 操作系统共能

### 处理机管理

由于操作系统对处理机管理策略的不同，其提供的作业处理方式也就不同；

+ 成批处理方式
+ 分时处理方式
+ 实时处理方式

### 存储管理

存储管理：对内部存储器进行**分配、保护和扩充**；

### 设备管理

+ 通道、控制器、输入输出设备的分配和管理；
+ 设备独立性；

### 信息管理

对系统的软件资源的管理；

+ 共享、保密、保护；
+ 保证系统安全可靠；

### 用户接口

+ **程序一级的接口**：**广义指令**供用户程序和其他系统程序调用，进行数据传输、文件操作或有其他资源要求时，通过这些广义指令向操作系统提出申请，并由操作系统代为完成。
+ **作业一级的接口**：**控制操作指令**供用户组织和控制自己作业的运行；
  + 作业控制方式两大类：脱机控制和联机控制；
  + 操作系统提供**脱机控制作业语言和联机控制作业控制语言**

## 计算机硬件

1. 操作系统管理和控制计算机系统中所有软硬件资源；
2. 操作系统是一个运行于硬件之上的系统软件；
